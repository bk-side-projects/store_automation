
# 프로젝트 청사진: Figma 기반 관리자 대시보드

## 개요

이 문서는 Figma 디자인을 기반으로 Next.js와 Firebase를 사용하여 구축하는 "통영아재수산" 관리자 대시보드 애플리케이션의 개발 계획, 과정, 그리고 최종 명세를 정의합니다. 이 문서는 단순한 계획을 넘어, 프로젝트가 겪은 문제와 해결 과정을 투명하게 기록하는 살아있는 역사가 될 것입니다.

---

## 1. 기반 구축 및 안정화 (완료)

- **환경 설정**: Next.js App Router 기반 프로젝트 설정 및 `.idx/mcp.json` 파일에 Firebase 서버 구성을 완료했습니다.
- **개발 서버 안정화**: `rm -rf .next` 명령으로 개발 서버 캐시를 초기화하여, 반복되던 런타임 오류를 해결하고 안정적인 개발 환경을 확보했습니다.
- **인증 시스템 확립**: `components/Providers.tsx`와 `components/AppLayout.tsx`를 통해 사용자의 로그인/로그아웃 및 세션 관리가 안정적으로 작동하도록 구현했습니다.

---

## 2. UI 완성 및 타입 표준화 (완료)

- **목표**: Figma 디자인을 코드 레벨까지 완벽하게 구현하고, 프로젝트 전체의 타입 시스템을 표준화하여 안정성을 확보합니다.
- **1단계: 핵심 페이지 UI 재구축 (완료)**
    - **작업 내용**: `app/dashboard/page.tsx`와 그 하위 컴포넌트들을 Figma 시안과 100% 일치하도록 재구축하여, 시각적으로 완벽한 상세 주문 관리 페이지를 완성했습니다.
- **2단계: 타입 시스템 리팩토링 (완료)**
    - **작업 내용**: `types/order.ts`를 중심으로 프로젝트의 데이터 타입을 표준화하고, 모든 관련 컴포넌트(`app/dashboard/page.tsx` 등)가 이 중앙 타입을 사용하도록 리팩토링했습니다. 이 과정에서 Firestore 연동 로직을 임시로 목업 데이터로 전환하여 안정적인 타입 리팩토링을 진행했습니다.
- **3단계: 메인 대시보드 및 로그인 페이지 리팩토링 (완료)**
    - **작업 내용**: `app/page.tsx`(메인 대시보드)와 `app/login/page.tsx`(로그인) 및 그 하위 컴포넌트(`StatCard`, `SalesChart`, `RecentOrdersTable`)들을 Figma 디자인 시스템에 맞춰 전면적으로 리팩토링하여, 프로젝트 전체의 시각적 일관성을 확보했습니다.

---

## 3. 데이터 소스 실시간화 (완료)

- **목표**: 애플리케이션 전체에서 사용하는 모든 데이터를 실시간 Firestore 데이터베이스로 통일하여, 모든 페이지가 동기화된 정보를 표시하도록 합니다.
- **작업 내용**: `lib/data.ts` 파일이 목업 데이터 대신 실제 Firestore의 `orders` 컬렉션을 조회하도록 성공적으로 리팩토링했습니다.

---

## 4. 주문 관리 핵심 기능 구현 (완료)

데이터 소스가 실시간으로 통일되었으므로, 이제 실제 관리 기능을 구현합니다.

1.  **새 주문 추가 (완료)**: '새 주문 추가' 버튼에 Firestore 문서 생성 기능을 Server Action으로 연결하여 성공적으로 구현했습니다.
2.  **주문 상태 변경 (완료)**: 테이블 내에서 각 주문의 상태를 동적으로 변경하고 Firestore 데이터를 업데이트하는 `StatusSelector` 컴포넌트를 구현했습니다.
3.  **주문 수정/삭제 (완료)**: `OrderActions` 컴포넌트와 `deleteOrder` 서버 액션을 통해 주문 삭제 기능을 구현하고, 수정 기능의 확장 기반을 마련했습니다.

---

## 5. UI/UX 기능 고도화 (완료)

관리 대시보드의 사용성 향상을 위해 검색, 필터링, 페이지네이션 등 고급 기능을 성공적으로 구현했습니다.

1.  **검색 기능 구현 (완료)**: 고객 이름으로 주문을 실시간으로 검색하는 기능을 성공적으로 구현했습니다.
2.  **필터링 기능 구현 (완료)**: 주문 상태별로 주문을 필터링하는 기능을 성공적으로 구현했습니다.
3.  **페이지네이션 구현 (완료)**: 많은 양의 주문 데이터를 페이지로 나누어 보여주는 기능을 성공적으로 구현했습니다.

---

## 6. 배포

이제 완성된 애플리케이션을 Firebase Hosting에 배포하여 최종 사용자가 접근할 수 있도록 합니다.
